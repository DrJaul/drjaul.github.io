Shadowrun Cyberdeck Helper

The Shadowrun (5e) Cyberdeck Configuration Helper solves the persistent table problem of juggling hardware stats, software suites, and situational modifiers in the heat of a run. It is written for game masters, deckers, and technomancers who need a quick way to balance legality, cost, and dice pools without rifling through multiple books mid-session. The helper guides players through configuring a legal or illicit deck step by step, keeps track of attack, sleaze, data processing, and firewall ratings, and flags potential issues before a plan goes live. Alongside numeric summaries, it offers interpretive copy that explains why specific loadouts shine for reconnaissance, brute-force intrusions, or surgical data heists. By centralizing calculations and reminders in an approachable interface, it frees the table to focus on fiction-forward decisions instead of spreadsheet wrangling. Every description is authored from scratch to keep the resource compliant with publisher policies.

AITree Tactical Growth Simulator

AITree expands the classic classroom min-max tree assignment into a full three-dimensional growth engine. The library models the full lifecycle of procedural trees on a voxel-style grid and exposes controller routines to push the simulation forward one branch at a time. It is designed as the behaviour and data core that other projects—like the companion Unity viewer—can plug into when they need authoritative growth logic. Because the repository follows a model-controller split, teams can extend the growth heuristics, run automated experiments, or feed the state into a renderer without rewriting the underlying systems.

AITree Display (Unity Viewer)

AITree Display is the Unity front end that turns the TreeAI simulation data into a living forest of procedural branches. Acting as the visual layer in an MVC workflow, the project synchronizes with the compiled TreeAI library, issues growth commands, and renders each iteration in real time. Developers clone both repositories, link the compiled TreeAI assembly, and jump into Play mode to watch the generated trees react to controller tweaks. By keeping the visualization logic isolated here, teams can keep experimenting with the simulation model while enjoying a responsive, cinematic viewer.
